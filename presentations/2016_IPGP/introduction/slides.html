What is ObsPy?

Provide complete Python framework to work with all types of seismological data

  • read/write file format support
    • waveforms (MiniSEED, SAC, GSE, SEISAN, SEG-Y, SEG2, Q, CSS, ...)
    • station metadata (StationXML, dataless/full SEED, RESP, SACPZ)
    • event metadata (QuakeML, NDK, NLLOC, ZMAP, CNV, MCHEDR)
      •
  • data center access (FDSN WS, ArcLink, SeedLink, Earthworm, NEIC, ...)
  • unified object classes (Waveforms, Events, Stations)
  • signal processing routines
    • trimming, merging, tapering, rotating, resampling, interpolation, ...
    • filtering, instrument correction, ...
    • array analysis, CC routines, PPSD, event detection, TF misfits, ...
      •

What's the beauty of it?

  • Runs on all Platforms
    • Windows, Mac, Linux, FreeBSD (Installers, Debian/Ubuntu repository, ...)
    • from RaspberryPi (35$) to SuperMUC (>150 Mio.$)
      •
  • Free and open source (LGPLv3)
  • Community pages: GitHub, wiki, tutorials, gallery, installation instructions, ticket system, mailing list
  • Well documented source code, auto-generated API docs
  • Test-driven dev. (>1200 tests, 86% Test Coverage, Travis/AppVeyor CI)
  • Reuses well established third-party code
    • Python community: NumPy, SciPy, matplotlib, ...
    • Seismo community: libmseed, evalresp, taup, GSE UTI, ...

  • Python!
